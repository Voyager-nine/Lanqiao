# 考点：LED 指示灯与二进制位映射

## 题目含义

题图的要求是：

- 用 `L1-L8` 这 8 个 LED 的亮灭状态表示测距结果。
- 8 个 LED 每个只有两种状态：亮 / 灭。
- 因此总共可以组合出 `2^8 = 256` 种状态。
- 当测距结果大于等于 `255cm` 时，要求 `L1-L8` 全亮。

题图给出的示例是：

- 当距离为 `90cm` 时，LED 状态为：`L2、L4、L5、L7` 亮。

这本质上是在考：

1. 十进制数转二进制。
2. 二进制每一位如何映射到 LED。
3. 程序里如何正确拆位。
4. 如何处理边界值 `255`。

## 为什么 90cm 会对应图中的状态

先把 `90` 转成 8 位二进制：

```c
90 = 01011010
```

把这 8 位从高位到低位写出来是：

- bit7 = 0
- bit6 = 1
- bit5 = 0
- bit4 = 1
- bit3 = 1
- bit2 = 0
- bit1 = 1
- bit0 = 0

题图中从左到右看到的是：

- L1 灭
- L2 亮
- L3 灭
- L4 亮
- L5 亮
- L6 灭
- L7 亮
- L8 灭

也就是：

```c
01011010
```

这说明这道题按视觉顺序定义时，采用的是：

- `L1 -> bit7`
- `L2 -> bit6`
- `L3 -> bit5`
- `L4 -> bit4`
- `L5 -> bit3`
- `L6 -> bit2`
- `L7 -> bit1`
- `L8 -> bit0`

也就是说：

- 最左边的 LED 显示最高位。
- 最右边的 LED 显示最低位。

这是这题最关键的位序关系。

## 这类题最容易犯的错误

### 1. 把 LED 顺序和位顺序弄反

很多人会默认写成：

- `L1 -> bit0`
- `L2 -> bit1`
- ...
- `L8 -> bit7`

这样写在某些数值下可能看起来也“像是对的”，但一旦拿题图示例去对，比如 `90`、`120`，就会发现和标准图不一致。

本题按图判断，应使用：

```c
L1 -> bit7
L2 -> bit6
L3 -> bit5
L4 -> bit4
L5 -> bit3
L6 -> bit2
L7 -> bit1
L8 -> bit0
```

### 2. 把位掩码当成 0/1 状态

错误写法：

```c
ucLed[i] = (Distance >> i) & (1 << i);
```

或者：

```c
ucLed[i] = Distance & (1 << i);
```

这样得到的不是纯粹的 `0` 或 `1`，而可能是：

- `0`
- `2`
- `4`
- `8`
- `16`

但很多 LED 驱动函数要求 `ucLed[i]` 只能是 `0` 或 `1`。

正确思路是：

- 先把目标位移到最低位。
- 再只取最低位。

正确写法：

```c
ucLed[i] = (Distance >> x) & 1;
```

### 3. 全亮边界值没有单独处理

题目明确规定：

- 测距结果 `>= 255cm` 时，全亮。

所以不能完全依赖普通拆位逻辑，应该单独判断：

```c
if (Distance >= 255)
{
    for(i = 0; i < 8; i++)
    {
        ucLed[i] = 1;
    }
}
```

## 这道题应该怎么做

### 第一步：先确定题目采用的位序

根据题图中 `90cm` 的示例：

```c
90 = 01011010
```

图中从 `L1` 到 `L8` 也正好是：

```c
01011010
```

所以这道题应采用：

```c
L1 -> bit7
L2 -> bit6
L3 -> bit5
L4 -> bit4
L5 -> bit3
L6 -> bit2
L7 -> bit1
L8 -> bit0
```

### 第二步：程序中按这个位序拆位

如果 `ucLed[0]` 对应 `L1`，`ucLed[7]` 对应 `L8`，那么应写成：

```c
for(i = 0; i < 8; i++)
{
    ucLed[i] = (Distance >> (7 - i)) & 1;
}
```

解释：

- 当 `i = 0` 时，`ucLed[0]` 取 `bit7`，对应 `L1`。
- 当 `i = 1` 时，`ucLed[1]` 取 `bit6`，对应 `L2`。
- ...
- 当 `i = 7` 时，`ucLed[7]` 取 `bit0`，对应 `L8`。

### 第三步：处理 `Distance >= 255` 的特殊要求

完整推荐写法：

```c
void Led_Proc()
{
    unsigned char i;

    switch(Seg_Show_Mode)
    {
        case 0:
            if(Distance >= 255)
            {
                for(i = 0; i < 8; i++)
                {
                    ucLed[i] = 1;
                }
            }
            else
            {
                for(i = 0; i < 8; i++)
                {
                    ucLed[i] = (Distance >> (7 - i)) & 1;
                }
            }
        break;
    }

    Led_Disp(ucLed);
}
```

## 为什么这里要用 `(7 - i)`

因为你要的是“正着显示”。

所谓“正着显示”，就是：

- `Distance = 120`
- 二进制写成：`01111000`
- 从左到右的 `L1-L8` 也希望看到：`01111000`

那么最左边的 `L1` 就必须对应最高位 `bit7`，而不是最低位 `bit0`。

所以才要写成：

```c
ucLed[i] = (Distance >> (7 - i)) & 1;
```

如果写成：

```c
ucLed[i] = (Distance >> i) & 1;
```

那就是另一种定义：

- `L1 -> bit0`
- `L8 -> bit7`

这种写法在“内部逻辑”上也没错，但和题图的视觉顺序不一致。

## 做这类题的通用步骤

以后遇到“LED 显示二进制状态”的题，建议固定按下面步骤做：

### 1. 先看题图给的样例

不要一开始就写代码，先拿题目给的样例反推位序。

重点看：

- 左边 LED 是高位还是低位。
- 右边 LED 是低位还是高位。
- LED 亮时逻辑值是 `1` 还是 `0`。

### 2. 先手算一个样例

比如题图给 `90cm`，先手算：

```c
90 = 01011010
```

再和图上的 LED 一一对应。

这样能先把位序搞准。

### 3. 再决定程序公式

如果题目要求：

- `L1 -> bit7`

就写：

```c
ucLed[i] = (Distance >> (7 - i)) & 1;
```

如果题目要求：

- `L1 -> bit0`

就写：

```c
ucLed[i] = (Distance >> i) & 1;
```

### 4. 最后处理特殊边界值

像本题这种：

- `Distance >= 255` 全亮

必须单独处理，不要混在普通逻辑里赌结果。

## 本题最终结论

本题按题图要求，推荐结论如下：

- `ucLed[0]` 对应 `L1`
- `ucLed[7]` 对应 `L8`
- 视觉顺序采用“高位在左、低位在右”
- 因此 LED 映射应为：`L1 -> bit7`，`L8 -> bit0`
- 正确拆位公式应为：

```c
ucLed[i] = (Distance >> (7 - i)) & 1;
```

- 当 `Distance >= 255` 时，直接让 `ucLed[0..7] = 1`

## 可直接套用的模板

```c
void Led_Proc()

    unsigned char i;

    switch(Seg_Show_Mode)
    {
        case 0:
            if(Distance >= 255)
            {
                for(i = 0; i < 8; i++)
                {
                    ucLed[i] = 1;
                }
            }
            else
            {
                for(i = 0; i < 8; i++)
                {
                    ucLed[i] = (Distance >> (7 - i)) & 1;
                }
            }
        break;
    }

    Led_Disp(ucLed);
}
```

## 如何理解 `ucLed[i] = (Distance >> x) & 1;`

这句代码的作用是：

- 从 `Distance` 的二进制表示中，取出第 `x` 位。
- 取出的结果只可能是 `0` 或 `1`。
- 这个结果刚好可以直接作为 LED 的灭 / 亮状态。

### 可以拆成两步理解

原式：

```c
ucLed[i] = (Distance >> x) & 1;
```

等价于先做这两步：

### 第一步：`Distance >> x`

把 `Distance` 右移 `x` 位。

目的：

- 让原来第 `x` 位移动到最低位。

例如：

```c
Distance = 90 = 01011010
```

如果 `x = 4`，那么：

```c
Distance >> 4
= 01011010 >> 4
= 00000101
```

原来的 `bit4` 已经被移到了最右边。

### 第二步：`& 1`

把右移后的结果和 `00000001` 做按位与。

作用：

- 只保留最低位。
- 其他位全部清零。

继续上面的例子：

```c
00000101 & 00000001 = 00000001
```

结果为：

```c
1
```

说明原来的 `bit4 = 1`。

如果最低位是 `0`，结果就会是 `0`。

## 为什么最后得到的一定是 `0` 或 `1`

因为任何数和 `1` 做按位与时，只会保留最低位：

- 最低位是 `1`，结果就是 `1`
- 最低位是 `0`，结果就是 `0`

所以：

```c
(Distance >> x) & 1
```

本质上就是在问：

- `Distance` 的第 `x` 位到底是 `0` 还是 `1`

这非常适合给 `ucLed[i]` 赋值。

## 为什么不能写成 `Distance & (1 << x)`

很多人容易写成：

```c
ucLed[i] = Distance & (1 << x);
```

这个写法不是“取出第 `x` 位并变成 `0/1`”，而是“保留第 `x` 位对应的掩码值”。

例如：

```c
Distance = 90 = 01011010
x = 4
```

则：

```c
Distance & (1 << 4)
= 01011010 & 00010000
= 00010000
= 16
```

结果是 `16`，不是 `1`。

但 `ucLed[i]` 一般需要的是：

- `0` 表示灭
- `1` 表示亮

所以应写成：

```c
ucLed[i] = (Distance >> x) & 1;
```

## 一句话记忆

`(Distance >> x) & 1` 的意思就是：

- 先把第 `x` 位移到最右边
- 再看看它是 `0` 还是 `1`

这就是二进制拆位最标准的写法。
